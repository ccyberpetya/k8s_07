# Домашнее задание «Компоненты Kubernetes»

## Расчёт ресурсов приложения

| Компонент   | Количество реплик | CPU на реплику | RAM на реплику |  CPU всего |    RAM всего |
| ----------- | ----------------: | -------------: | -------------: | ---------: | -----------: |
| База данных |                 3 |              1 |           4 ГБ |          3 |        12 ГБ |
| Кеш         |                 3 |              1 |           4 ГБ |          3 |        12 ГБ |
| Фронтенд    |                 5 |            0,2 |          50 МБ |          1 |       250 МБ |
| Бекенд      |                10 |              1 |         600 МБ |         10 |         6 ГБ |
| **Итого**   |            **21** |                |                | **17 CPU** | **30,25 ГБ** |

Расчёт:

```text
База данных:
3 × 1 CPU = 3 CPU
3 × 4 ГБ = 12 ГБ

Кеш:
3 × 1 CPU = 3 CPU
3 × 4 ГБ = 12 ГБ

Фронтенд:
5 × 0,2 CPU = 1 CPU
5 × 50 МБ = 250 МБ

Бекенд:
10 × 1 CPU = 10 CPU
10 × 600 МБ = 6000 МБ = 6 ГБ

Всего:
17 CPU
30,25 ГБ RAM
```

## Расчёт ресурсов Kubernetes-кластера

Для worker-ноды выбрана конфигурация:

```text
8 CPU
16 ГБ RAM
```

С учётом резерва на Kubernetes и операционную систему для Pod остаётся примерно:

```text
7 CPU
14 ГБ RAM
```

Для обеспечения отказоустойчивости потребуется четыре worker-ноды.

После отказа одной worker-ноды останется:

```text
21 CPU
42 ГБ RAM
```

Этого достаточно для приложения, которому требуется 17 CPU и 30,25 ГБ RAM.

## Итоговая конфигурация

* 3 control-plane-ноды: 2 CPU и 4 ГБ RAM;
* 4 worker-ноды: 8 CPU и 16 ГБ RAM.

Всего: **7 нод, 38 CPU и 76 ГБ RAM**.
